Side-by-side

What gets shorter, and what stays explicit.

Each pair below shows the same behavior with less ceremony.

Storage

Same counter update, less storage plumbing

Manual pattern

#[derive(Clone)]
#[contracttype]
enum DataKey {
    Value,
}

pub fn increment_by(env: &Env, amount: u32) -> u32 {
    let current = env
        .storage()
        .instance()
        .get(&DataKey::Value)
        .unwrap_or(0);

    let next = current.saturating_add(amount);
    env.storage().instance().set(&DataKey::Value, &next);
    next
}

With #[contractstorage]

#[contractstorage]
pub struct Counter {
    value: InstanceItem<u32>,
}

pub fn increment_by(env: &Env, amount: u32) -> u32 {
    Counter::update_value(env, |current| {
        current.unwrap_or(0).saturating_add(amount)
    })
}

Both versions increment the same counter and return the result. The generated version drops the boilerplate.

Errors

Same safe_div, less error maintenance

Manual pattern

#[contracterror]
#[derive(Copy, Clone, Eq, PartialEq)]
#[repr(u32)]
pub enum CalcError {
    DivisionByZero = 100,
    NegativeInput = 200,
}

pub fn safe_div(num: i64, denom: i64) -> Result<i64, CalcError> {
    if denom == 0 {
        return Err(CalcError::DivisionByZero);
    }
    if denom < 0 {
        return Err(CalcError::NegativeInput);
    }
    Ok(num / denom)
}

With #[scerr]

#[scerr]
pub enum MathError {
    /// division by zero
    DivisionByZero,
    NegativeInput,
}

pub fn safe_div(num: i64, denom: i64) -> Result<i64, MathError> {
    if denom == 0 {
        return Err(MathError::DivisionByZero);
    }
    if denom < 0 {
        return Err(MathError::NegativeInput);
    }
    Ok(num / denom)
}

Both functions do the same checks and return the same result. The difference is how the errors are maintained.

Auth testing

Same token transfer, less auth setup

Manual MockAuth

env.mock_auths(&[
    soroban_sdk::testutils::MockAuth {
        address: &alice,
        invoke: &soroban_sdk::testutils::MockAuthInvoke {
            contract: &contract_id,
            fn_name: "transfer",
            args: (&alice, &bob, 300_i128).into_val(env),
            sub_invokes: &[],
        },
    },
]);

client.transfer(&alice, &bob, &300);

Generated AuthClient

token::AuthClient::new(env, &contract_id)
    .transfer(&alice, &bob, &300)
    .authorize(&alice)
    .invoke();

Both tests authorize the same transfer. The builder version removes the nested auth structure.

Reduction strategy

Code gets smaller where it should.

The goal is simple: less repetition around storage, errors, and auth.

Net effect

  • Fewer copy-paste mistakes
  • More readable examples
  • Cleaner auth tests
