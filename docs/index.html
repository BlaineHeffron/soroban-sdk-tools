Core surface area

Three places Soroban projects pick up drag.

Storage

Typed accessors instead of key enums

Define storage once. The reads, writes, and updates come along with it.

Errors

Composable errors with predictable codes

#[scerr] handles sequential enums and anchored ranges like 100-199 without extra glue.

Auth testing

Auth flows that read like the call

AuthClient turns nested auth setup into something you can read at a glance.

Why this project exists

Soroban gets easier when onboarding gets shorter.

Rust developers should be able to start writing Soroban contracts without building their own macro layer first.

The SDK still looks like the SDK. You just spend less time on the repetitive parts.

Good fit

  • Teams standardizing storage across contracts
  • Projects composing errors across boundaries
  • Test suites with repeated auth setup

Design principle

Remove repetition, not control. Your contract code stays explicit.
