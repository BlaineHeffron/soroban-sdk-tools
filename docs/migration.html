Incremental adoption

Migrate one concern at a time.

No rewrite needed. Start with storage, then errors, then auth tests when you get there.

Recommended sequence

  1. Start with #[contractstorage].
  2. Replace manual error codes with #[scerr].
  3. Move from manual MockAuth to AuthClient.
  4. Use real signer helpers only when signatures matter.

Step 1

Storage migration

What to replace

Manual key enums, repeated env.storage() calls, and helper code you've probably copied more than once.

What to introduce

A storage struct with #[contractstorage]. Add auto_shorten = true if you want shorter keys.

Checklist

  • Move keys into one struct.
  • Choose persistent, instance, or temporary storage explicitly.
  • Replace custom helpers with generated methods like get_field and set_field.
  • Use Storage::new(env) when grouped access reads better.

Watch for

  • Key names are part of your storage layout.
  • Changing durability or key shape is still a migration.
  • Be careful with auto_shorten on live contracts.

Step 2

Error migration

What to replace

Manual error codes and the conversion glue that comes with them.

What to introduce

#[scerr], plus transparent or client variants where needed.

Checklist

  • Convert one enum at a time.
  • Put descriptions in doc comments.
  • Use #[transparent] with #[from] for same-WASM composition.
  • Use #[from_contract_client] for client try_ calls.

Watch for

  • Enum order defines the generated codes.
  • Cross-contract composition depends on imported error support.
  • Keep old numeric mappings documented on live surfaces.

Step 3

Auth test migration

What to replace

Repeated MockAuth arrays and nested invoke trees that are hard to read six weeks later.

What to introduce

AuthClient, plus setup_mock_auth and setup_real_auth when needed.

Checklist

  • Import the contract WASM with contractimport!.
  • Create an AuthClient alongside the normal client.
  • Replace manual auth trees with chained .authorize() calls.
  • Use .sign() only when signature behavior matters.

Watch for

  • Mock auth is still the default for most tests.
  • The readability gain is biggest in multi-signer flows.
  • Save real signer tests for signature semantics.

Migration principle

Remove repetition, not domain logic.

Keep storage, invariants, and auth rules visible. Cut the boilerplate around them.
